# 实现计划
## 小学生作业打卡系统 - V1.0 MVP

---

## 一、总览

基于 SDD，将实现分为 **6 个阶段**，按依赖顺序排列。每个阶段可独立验证，确保每步都有可运行的结果。

| 阶段 | 内容 | 预计工时 |
|------|------|----------|
| P0 | 项目脚手架 & 工具链 | 1h |
| P1 | 数据库 & 后端 API | 3h |
| P2 | 前端核心框架 | 2h |
| P3 | 首页 & 打卡功能 | 3h |
| P4 | 任务管理（家长端） | 2h |
| P5 | 统计页面 & 收尾 | 2h |
| **合计** | | **~13h** |

## 二、阶段详细计划

### P0：项目脚手架 & 工具链

**目标：** 项目可 `npm run dev` 启动，前后端基础结构就绪。

#### 步骤

1. **初始化项目**
   - 创建 `package.json`
   - 安装依赖：
     - 运行时：`express`, `better-sqlite3`, `multer`, `date-fns`, `uuid`
     - 前端：`react`, `react-dom`, `react-router-dom`, `chart.js`, `react-chartjs-2`
     - 开发：`typescript`, `vite`, `@vitejs/plugin-react`, `tailwindcss`, `postcss`, `autoprefixer`, `tsx`, `concurrently`, `nodemon`
     - 类型：`@types/express`, `@types/react`, `@types/react-dom`, `@types/multer`, `@types/better-sqlite3`, `@types/uuid`

2. **配置文件**
   - `tsconfig.json`（服务端，target ES2020, commonjs）
   - `tsconfig.client.json`（客户端，target ES2020, ESNext module, JSX）
   - `vite.config.ts`（代理 `/api` 到 Express，输出到 `dist/client`）
   - `tailwind.config.js`（content 扫描 `src/client/**/*.{tsx,html}`）
   - `postcss.config.js`
   - `.env`（`PORT=3000`, `DATABASE_URL="file:./homework.db"`）

3. **目录结构**
   - 创建所有目录（`src/server/routes`, `src/client/{components,pages,hooks,types}`）

4. **入口文件**
   - `src/server/index.ts`：Express 监听端口，生产模式 serve 静态文件
   - `src/server/app.ts`：中间件配置（cors, json, static uploads）
   - `src/client/index.html`：含 `<div id="root">` 的 HTML 壳
   - `src/client/main.tsx`：ReactDOM.createRoot + 引入 Tailwind CSS

5. **package.json scripts**
   ```json
   {
     "dev": "concurrently \"npm run dev:server\" \"npm run dev:client\"",
     "dev:server": "tsx watch src/server/index.ts",
     "dev:client": "vite",
     "build": "vite build && tsc -p tsconfig.json",
     "start": "node dist/server/index.js"
   }
   ```

6. **验证：** `npm run dev` 后浏览器打开，页面显示 "Hello"，API 访问 `GET /api/health` 返回 `{ "ok": true }`。

### P1：数据库 & 后端 API

**目标：** 数据库就绪，所有 API 接口可调用。

#### 步骤

1. **Prisma Schema**
   - 文件：`prisma/schema.prisma`
   - 定义三张表：`Task`, `CheckIn`, `Setting`（按 SDD 4.2 定义字段）
   - `npx prisma generate` 生成 Prisma Client
   - `npx prisma db push` 生成 SQLite 数据库

2. **数据库连接**
   - `src/server/db.ts`：导出 Prisma Client 单例

3. **任务路由** — `src/server/routes/tasks.ts`
   - `GET /api/tasks` — 查询所有 `isActive=1` 的任务，按 `sortOrder` 排序
   - `GET /api/tasks/today` — 调用 `getTodayTasks()` 逻辑，附带每个任务的今日打卡状态
   - `GET /api/tasks/:id` — 单任务查询
   - `POST /api/tasks` — 创建（校验 name/subject 必填）
   - `PUT /api/tasks/:id` — 更新
   - `DELETE /api/tasks/:id` — 软删除（`isActive = 0`）
   - `PUT /api/tasks/:id/sort` — 排序

4. **打卡路由** — `src/server/routes/checkins.ts`
   - `POST /api/checkins` — 打卡（检查重复打卡，记录 completedAt）
   - `DELETE /api/checkins/:id` — 取消打卡
   - `GET /api/checkins/date/:date` — 按日期查询
   - `GET /api/checkins/range` — 日期范围查询

5. **统计路由** — `src/server/routes/stats.ts`
   - `GET /api/stats/overview` — 今日/本周完成率 + 连续天数
   - `GET /api/stats/completion-rate` — 按 period=week|month 返回趋势数据
   - `GET /api/stats/subject` — 各科目完成数/总数
   - `GET /api/stats/calendar` — 指定年月的日历打卡数据
   - `GET /api/stats/streak` — 当前连续 + 最长连续

6. **上传路由** — `src/server/routes/upload.ts`
   - `POST /api/upload` — multer 中间件，限制 jpg/png/webp、≤5MB，文件名用 UUID
   - 静态服务：`app.use('/uploads', express.static('uploads'))`

7. **设置路由** — `src/server/routes/settings.ts`
   - `POST /api/settings/verify` — 密码比对
   - `PUT /api/settings/password` — 修改密码

8. **app.ts 路由注册**
   - 将全部路由挂载到 `app.use('/api/tasks', taskRouter)` 等

9. **验证：** 用 curl / Postman 测试所有 CRUD 接口正常。

**curl 测试示例：**
```bash
# 创建任务
curl -X POST http://localhost:3000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"name":"数学口算","subject":"数学","emoji":"🔢","estimatedMin":10,"repeatType":"daily"}'

# 获取今日任务
curl http://localhost:3000/api/tasks/today

# 打卡
curl -X POST http://localhost:3000/api/checkins \
  -H "Content-Type: application/json" \
  -d '{"taskId":1,"quality":3}'
```

### P2：前端核心框架

**目标：** React 应用可运行，路由和布局就绪。

#### 步骤

1. **类型定义** — `src/client/types/index.ts`
   ```typescript
   interface Task {
     id: number;
     name: string;
     subject: string;
     emoji: string;
     estimatedMin: number;
     deadlineTime: string | null;
     repeatType: 'once' | 'daily' | 'weekly';
     repeatDays: number[];
     sortOrder: number;
     isActive: boolean;
     createdAt: string;
     updatedAt: string;
   }

   interface CheckIn {
     id: number;
     taskId: number;
     date: string;
     completedAt: string;
     quality: number | null;
     photoUrl: string | null;
     note: string | null;
     isMakeup: boolean;
   }

   interface TaskWithStatus extends Task {
     isCheckedIn: boolean;
     checkInId: number | null;
     quality: number | null;
   }
   ```

2. **AppContext** — `src/client/App.tsx`
   - 创建 Context，管理 `tasks`, `todayCheckins`, `stats`, `isParentMode`
   - 提供 `refreshData()` 方法
   - 全局 Layout：顶部日期 + 连续天数徽章 + 底部导航

3. **路由配置**
   - `/` → HomePage
   - `/tasks` → TasksPage
   - `/stats` → StatsPage
   - `/settings` → SettingsPage

4. **NavBar 组件** — `src/client/components/NavBar.tsx`
   - 底部固定导航栏（移动端优先）
   - 四个图标：🏠 📋 📊 ⚙️
   - 当前页面高亮

5. **Layout 组件** — `src/client/components/Layout.tsx`
   - 响应式容器：max-width 800px，居中
   - 底部留出 NavBar 空间

6. **自定义 Hooks**
   - `src/client/hooks/useTasks.ts`：封装任务 API 调用
   - `src/client/hooks/useCheckins.ts`：封装打卡 API 调用
   - `src/client/hooks/useStats.ts`：封装统计 API 调用

7. **验证：** 页面可切换路由，NavBar 正常显示/高亮。

### P3：首页 & 打卡功能

**目标：** 孩子可以查看今日任务并完成打卡。

#### 步骤

1. **HomePage** — `src/client/pages/HomePage.tsx`
   - 顶部：当天日期（"5月25日 星期一"）
   - 连续打卡 StreakBadge 展示
   - TaskList 组件
   - 页面加载时调用 `GET /api/tasks/today`

2. **TaskList 组件** — `src/client/components/TaskList.tsx`
   - 遍历渲染 TaskItem
   - 空状态：显示 "今天没有任务，休息一下吧 🎉"

3. **TaskItem 组件** — `src/client/components/TaskItem.tsx`
   - 左侧：emoji + 任务名 + 科目标签
   - 右侧：预计时长 + CheckInButton / 已完成状态
   - 已完成：绿色勾选 + 质量星星

4. **CheckInButton 组件** — `src/client/components/CheckInButton.tsx`
   - 未打卡：大按钮 "完成 ✓"（主色调，圆角，大字号 ≥16px）
   - 已打卡：灰色 "已完成 ✅"
   - 点击后：
     1. 乐观更新 UI
     2. 调用 `POST /api/checkins`
     3. 成功：撒花动画（CSS animation）
     4. 失败：回滚 UI，toast 提示

5. **StreakBadge 组件** — `src/client/components/StreakBadge.tsx`
   - 显示 "🔥 连续 X 天"
   - 调用 `GET /api/stats/streak`

6. **QualityStars 组件** — `src/client/components/QualityStars.tsx`
   - 三档星星选择（打卡时可标记）
   - 大号星星，适合孩子点击

7. **验证：** 创建任务 → 首页显示 → 点击完成 → 显示已打卡 + 动画。

### P4：任务管理（家长端）

**目标：** 家长可创建、编辑、删除、排序任务。

#### 步骤

1. **TasksPage** — `src/client/pages/TasksPage.tsx`
   - 密码验证守卫（检查 localStorage `parentMode`）
   - 未验证：显示密码输入框
   - 已验证：显示任务管理界面

2. **TaskForm 组件** — `src/client/components/TaskForm.tsx`
   - 弹窗/抽屉形式
   - 字段：
     - 任务名（文本输入）
     - Emoji 选择（预设 📖🔢🔬🎨🎵📚 等）
     - 科目（下拉选择：语文/数学/英语/科学/其他）
     - 预计时长（数字，分钟）
     - 截止时间（时间选择器）
     - 重复类型（单选：一次性/每天/每周）
     - 每周重复时：星期多选（一二三四五六日）
   - 保存 / 取消按钮

3. **任务列表（管理视图）**
   - 显示所有任务（含已停用）
   - 每个任务：编辑按钮 + 删除按钮
   - 可拖拽排序（或上移/下移按钮）

4. **任务模板快捷创建**
   - 预设模板按钮：
     - 📖 语文朗读（10分钟，每日）
     - 🔢 数学口算（10分钟，每日）
     - 📝 英语背诵（15分钟，每日）
   - 点击直接创建

5. **验证：** 密码进入 → 创建/编辑/删除任务 → 首页同步更新。

### P5：统计页面 & 收尾

**目标：** 统计数据可视化，响应式完善，整体打磨。

#### 步骤

1. **StatsPage** — `src/client/pages/StatsPage.tsx`
   - 顶部：概览卡片（今日完成率、本周完成率、连续天数、总打卡次数）
   - 中部：月度日历热力图
   - 下部：完成率趋势折线图 + 科目柱状图

2. **StatsOverview 组件** — `src/client/components/StatsOverview.tsx`
   - 4 个卡片，Grid 布局
   - 调用 `GET /api/stats/overview`

3. **CheckInCalendar 组件** — `src/client/components/CheckInCalendar.tsx`
   - 月度日历网格
   - 有打卡的日期高绿色，未打卡灰色
   - 可切换月份
   - 调用 `GET /api/stats/calendar`

4. **StatsChart 组件** — `src/client/components/StatsChart.tsx`
   - 折线图：日/周完成率趋势
   - 柱状图：各科目完成情况
   - 使用 react-chartjs-2

5. **SettingsPage** — `src/client/pages/SettingsPage.tsx`
   - 修改家长密码
   - 关于信息

6. **响应式适配**
   - 验证三个断点（手机/平板/桌面）
   - 字体 ≥16px（孩子可读性）
   - 触摸目标 ≥44px

7. **验证：** 完成打卡后，统计数据正确更新，图表正常渲染。

  └─→ P1（后端 API）

## 四、验收标准

### V1.0 MVP 验收清单

| # | 验收项 | 验证方式 |
|---|--------|----------|
| 1 | 可创建任务（名称、科目、重复类型） | 创建后在列表显示 |
| 2 | 可编辑/删除任务 | 编辑后数据更新，删除后不显示 |
| 3 | 首页显示今日任务 | 按重复规则正确过滤 |
| 4 | 一键打卡（≤2次点击） | 点击后状态变"已完成" |
| 5 | 打卡成功动画 | 可见的庆祝效果 |
| 6 | 不能重复打卡同一任务 | 已打卡任务显示禁用状态 |
| 7 | 完成率统计正确 | 手动计算与显示一致 |
| 8 | 连续打卡天数正确 | 断开后重新计算 |
| 9 | 家长密码保护 | 未输密码无法进入任务管理 |
| 10 | 响应式适配 | 手机/平板/桌面布局正常 |
| 11 | 字体 ≥16px | 孩子可轻松阅读 |
